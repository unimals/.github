# Contributing to Unimals

Thanks for contributing.

## Working style

We value:

- small, focused pull requests
- clear problem statements
- practical solutions over unnecessary complexity
- documentation updates when behavior changes

## Before opening a PR

- check for existing issues or discussions
- keep scope tight
- explain the motivation and tradeoffs
- include verification steps

## PR expectations

- one clear purpose per PR
- readable title and summary
- notes on risks or follow-up work
- be responsive to review feedback

## Communication

Be direct, kind, and constructive.
